## Code Style & Strict Rules

TypeScript: 2 spaces, single quotes, no semicolons, trailing commas.

Node.js APIs in Workers are subset-only — verify compatibility before use:
https://developers.cloudflare.com/workers/runtime-apis/nodejs/

Cloudflare APIs change fast — always fetch current docs before any Workers/KV/R2/D1/DO/Queues/Vectorize/AI/Agents task.
Base docs URL: https://developers.cloudflare.com/workers/
Limits always live at: `/<product>/platform/limits/` (e.g. `/workers/platform/limits/`)

## Anti-Patterns & Restrictions

- Never manually edit `worker-configuration.d.ts` — auto-generated by `npm run cf-typegen`
- Never run test or lint scripts — none exist; don't guess
- Never hardcode secrets in source or config files
- Never assume wrangler.toml — this project uses wrangler.jsonc

## Security & Compliance

Secrets via `wrangler secret put <KEY>` only — never in source, wrangler.jsonc, or .env committed to git.

## Repository Quirks & Gotchas

- Config file is `wrangler.jsonc` (JSONC, not TOML) — do not create wrangler.toml
- Run `npm run cf-typegen` immediately after any binding change in wrangler.jsonc
- Error 1102 = CPU/memory exceeded → check `/workers/platform/limits/` before optimizing
- All errors reference: https://developers.cloudflare.com/workers/observability/errors/
